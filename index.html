root/overview/visualize-ai

Visualising the Invisible

A generative AI-powered learning platform that transforms abstract computer science concepts into interactive visual simulations — making the invisible visible for students who can read about processes but cannot see them.

What this is

The Problem

Abstract CS concepts — process scheduling, boolean logic, web crawling, confusion matrices — exist only as text and static diagrams in lecture slides. Students memorise definitions without ever seeing the concept behave. This gap between knowing and understanding is what VisualizeAI closes.

Objectives

Visualise abstract concepts

Use Gemini Canvas and NotebookLM to generate interactive, step-by-step simulations of concepts that students historically find hardest to understand.

Enable active learning

Move students from passive readers to active explorers — curating sources, synthesizing knowledge, and querying an AI tutor in real time.

Measure cognitive load

Surface where students struggle using real analytics — Conceptual Friction Index, Time-to-Resolution, and Syllabus Heatmap — so instructors can intervene before exam time.

root/module_01/conceptual-visualization

Module 1 — Conceptual Visualization

Instructors design structured pedagogical sequences to translate invisible concepts into visual, step-by-step simulations. Built with NotebookLM (knowledge consolidation) and Gemini Canvas (simulation generation), grounded in Cognitive Theory of Multimedia Learning and Cognitive Load Theory.

STATUS: READY TOOL: NOTEBOOKLM + GEMINI CANVAS TIER 1
Simulations by Subject
SUBJ.01Web Based Application Development ▾

Concept visualised: HTTP Request Lifecycle & Client-Server Architecture

Why it was invisible: Students read about GET and POST requests but cannot see the actual packet journey — what happens between typing a URL and a page appearing.

Simulation: Interactive step-through of an HTTP request from browser to server and back, showing headers, status codes, and state changes at each hop.

SIM.01Open Simulation ↗ sites.google.com/umt.edu.my/webapp2026k2

SUBJ.02Cybersecurity ▾

Concept visualised: Keselamatan Siber — Threats, Attack Vectors & Defense Mechanisms

Why it was invisible: Students learn about attacks and defenses as abstract terms without seeing how an exploit actually unfolds step-by-step or how a defense mechanism intercepts it.

Simulation: Interactive cybersecurity simulation walking through a threat scenario and the corresponding defense response in sequence.

SIM.02Open Simulation ↗ sites.google.com/umt.edu.my/csf3233

SUBJ.03Operating System ▾

Concept visualised: Process Scheduling & Memory Management

Why it was invisible: CPU scheduling happens in microseconds — without a simulation, students guess at round-robin and priority algorithms without seeing the decision logic execute.

Simulation: Interactive simulation linking process states to scheduling decisions, stepping through Chapter 1 concepts live.

SIM.03Open Simulation ↗ sites.google.com/umt.edu.my/operatingsystemnurul

SUBJ.04Information Retrieval ▾

Concept visualised: Web Crawling, Indexing & Ranking (Spider Bot)

Why it was invisible: A spider bot traverses millions of pages in milliseconds — the decision sequence at each node is invisible without a simulation to step through.

Simulation: Interactive Spider Bot — students trigger the crawl, watch indexing decisions happen in sequence, and see why certain pages rank before others.

SIM.04Open Simulation ↗ sites.google.com/umt.edu.my/spiderbotsimulator

root/module_02/ai-cocreation-assessments

Module 2 — AI Co-Creation Assessment

Students form groups of up to three to curate 5–12 credible technical sources. NotebookLM synthesizes these into structured evidence maps, reports, summaries, and citations — with each group member assigned a specific role in the process.

STATUS: IN PROGRESS TOOL: NOTEBOOKLM TIER 1
NotebookLM Synthesis Workflow
01

Consolidate Sources

Students curate 5–12 credible technical sources (journals, documentation, verified articles) as input files.

02

Information Synthesis

NotebookLM processes all sources, extracts key concepts and relationships, and builds a knowledge model.

03

Prompt Execution

Structured prompts guide generation of specific artifact types — structured reports, evidence maps, summaries, annotated citations.

04

Manual Verification

The Verifier role reviews all outputs against source material using verification sheets before submission.

05

Artifact Extraction

Final outputs — structured report, evidence map, source summary, citation list — ready for Presenter to deliver.
